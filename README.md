# Project C

Проект на языке C для сравнения двух способов сжатия изображений: JPEG-подобного сжатия и SVD-сжатия.

## Описание

Данный проект реализует обработку и сжатие изображений в формате BMP.

В программе используются два подхода:

1. JPEG-подобное сжатие.
2. SVD-сжатие.

JPEG-подобное сжатие работает с изображением блоками 8x8. Для каждого блока применяется дискретное косинусное преобразование, затем квантование и обратное преобразование.

SVD-сжатие основано на сингулярном разложении матрицы изображения. Изображение переводится в оттенки серого и восстанавливается по ограниченному количеству сингулярных значений.

После обработки программа сравнивает качество восстановленных изображений с оригиналом с помощью метрик PSNR и SSIM. Результаты сохраняются в изображения и в CSV-файл.

## Основной функционал

### Работа с изображениями

Проект умеет:

- читать BMP-файлы;
- сохранять BMP-файлы;
- создавать копию исходного изображения;
- обрабатывать RGB-данные изображения;
- сохранять результаты в папку `output`.

Основная работа с изображениями находится в папке `image`.

Файлы:

- `image/image.c`
- `image/image.h`

## JPEG-подобное сжатие

В проекте реализован упрощённый вариант JPEG-сжатия.

Основные этапы JPEG-подобного сжатия:

1. Разбиение изображения на блоки 8x8.
2. Получение значений яркости пикселей.
3. Применение DCT — дискретного косинусного преобразования.
4. Квантование коэффициентов.
5. Обратное DCT.
6. Сохранение восстановленного изображения.

Для JPEG-подобного сжатия используются разные значения параметра качества:

- `q = 1`
- `q = 2`
- `q = 4`
- `q = 8`

Чем больше значение `q`, тем сильнее сжатие, но тем ниже качество изображения.

Результаты JPEG-подобного сжатия сохраняются в файлы:

- `output/jpeg_q1.bmp`
- `output/jpeg_q2.bmp`
- `output/jpeg_q4.bmp`
- `output/jpeg_q8.bmp`
- `output/jpeg_like.bmp`

Файлы, связанные с JPEG-сжатием:

- `JPEG/dct.c`
- `JPEG/dct.h`
- `JPEG/jpeg.c`
- `JPEG/jpeg.h`

## SVD-сжатие

Также в проекте реализовано сжатие изображения с помощью SVD.

SVD — это сингулярное разложение матрицы. В данном проекте оно используется для приближённого восстановления изображения по ограниченному количеству компонент.

Программа проверяет несколько значений `k`:

- `k = 5`
- `k = 10`
- `k = 20`
- `k = 30`

Чем больше `k`, тем лучше качество восстановленного изображения, но тем меньше степень сжатия.

Результаты SVD-сжатия сохраняются в файлы:

- `output/svd_k5.bmp`
- `output/svd_k10.bmp`
- `output/svd_k20.bmp`
- `output/svd_k30.bmp`
- `output/svd.bmp`

В текущей реализации SVD применяется только для изображений размером не больше `160x160`.

Файлы, связанные с SVD:

- `SVD/svd.c`
- `SVD/svd.h`
- `SVD/jacobi.c`
- `SVD/jacobi.h`

## Метрики качества

Для сравнения качества сжатых изображений используются две метрики:

- PSNR
- SSIM

### PSNR

PSNR показывает, насколько восстановленное изображение отличается от оригинала. Чем выше значение PSNR, тем лучше качество изображения.

### SSIM

SSIM оценивает структурное сходство двух изображений. Значение ближе к 1 означает, что изображения очень похожи.

Файлы с метриками:

- `metrics/metrics.c`
- `metrics/metrics.h`

## Сборка проекта

Для сборки используется `Makefile`.

Чтобы собрать основную программу, нужно выполнить:

```bash
make
```

После этого будет создан исполняемый файл:

```text
program
```

Также можно явно выполнить:

```bash
make program
```

## Запуск программы

Запуск с изображением по умолчанию:

```bash
make run
```

В этом случае программа использует файл:

```text
input/test.bmp
```

Также можно запустить программу вручную:

```bash
./program
```

Или передать путь к своему BMP-файлу:

```bash
./program input/test.bmp
```

## Запуск тестов

Для сборки и запуска тестов используется команда:

```bash
make test
```

После этого будет создан и запущен файл:

```text
test_program
```

## Очистка проекта

Чтобы удалить собранные файлы и результаты работы программы, можно выполнить:

```bash
make clean
```

Эта команда удаляет:

- `program`;
- `test_program`;
- файлы `.bmp` из папки `output`;
- файлы `.csv` из папки `output`.


## Краткий принцип работы

1. Программа загружает BMP-изображение.
2. Сохраняет копию исходного изображения.
3. Выполняет JPEG-подобное сжатие с разными параметрами качества.
4. Выполняет SVD-сжатие с разными значениями `k`.
5. Для каждого результата считает PSNR и SSIM.
6. Сохраняет обработанные изображения в папку `output`.
7. Записывает итоговую таблицу сравнения в `output/results.csv`.
